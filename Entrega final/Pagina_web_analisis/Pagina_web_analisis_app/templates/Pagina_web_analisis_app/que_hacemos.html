¿Qué hacemos?

Nosotros desarrollamos una página web donde se puede observar el análisis de las iniciativas de ley en México. En este análisis, se comparan las iniciativas de ley por sexenio, trimestre, partido y ley más modificada. También, se analizan los casos de iniciativas aprobadas y desechadas. Además, se implementó un análisis con la ayuda del Term Frequency / Inverse Document Frequency (tf/idf por sus siglas en inglés), mismo que se presenta en la página. Por otra parte, se habilitó para los usuarios capacitados un página en la que se puede realizar un TF/IDF sobre las iniciativas, esto, para facilitar la búsqueda de iniciativas relevantes. Del mismo modo, implementamos un buscador de propuestas él cual es accesible para todos los usuarios.


Diagrama de flujo del desarrollo web

                             

Es necesario saber, que todos los datos que tenemos en nuestro desarrollo provienen del Apilegislativo. Es nuestra fuente de datos, la cual solo se usa para la extracción y almacenamiento de la misma.


Explicación de los procesos:


  1. Se consulta el Apilegislativo para así conocer los datos que este contiene, delimitando, nada más los necesarios para nuestro análisis.

  2. Después de haber seleccionado y extraído los datos, los guardamos en nuestra propia base de datos. Un API de Azure Cosmos DB para MongoDB el cual vive en el este de US, en los servidores de Azure.

  3. Con un código desarrollado en Python realizamos todo el análisis, para así poder obtener las gráficas.

  4. Así, llegamos al punto en donde nos encontramos, desarrollamos una página web en Django, una librería de Python. También, con ayuda de HTML, CSS y Bootstrap, le damos diseño y vida a la página.

  5. Nuestra página web va a vivir en el Centro de EE. UU. en los servidores de Azure. Gracias a los app services de Azure, nuestra página va a tener soporte suficiente para ejecutarse y ser consultada, sin saturarse.

  6. Por último, contamos con otro servicio de la familia Azure, hablamos de un Datalake que vive en el Este de EE. UU. en los servidores de Azure. Este es el encargado de almacenar todos los resultados de las consultas del TF/IDF.
